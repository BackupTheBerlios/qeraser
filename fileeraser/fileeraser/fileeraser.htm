Vorwort:

Das einrichten von Perlscripts sorgt vor allem bei Anfängern häufig für Verwirrung. Dabei muss man doch eigentlich nur ein paar Grundprinzipien verstehen:

Die meisten Server die Perlscripts unterstützen laufen unter einem Linuxsystem, daher muss man falls das script zuvor auf einer anderen Plattform, wie z. B Windows oder Mac bearbeitet wurde das Script auf jeden Fall im ASCII Modus hochladen, damit die systemabhängigen Zeilenumbrüche korrekt interpretiert werden können.
Genauso wichtig ist es wegen der Zugriffssicherheit der Linuxsysteme die Zugriffsrechte der Scripts richtig zu setzen.

Eine genaue Beschreibung zur Einrichtung von Perlscripts folgt unter den nächsten Punkten.

Scripte anpassen:

Vor dem Upload sollten die Scripts für den jeweiligen Server angepasst werden!

Die erste Zeile jedes Scriptes ist wichtig für den Betrieb auf Linuxsystemen. Sie gibt den serverinternen Pfad zum Perlinterpreten an. Meistens funktioniert mindestens eine der folgenden Möglichkeiten:
#!/usr/bin/perl
oder
#!/usr/local/bin/perl

Pfadangaben:
Ihr Host sollte Ihnen auf seiner Website Informationen zu den nötigen Pfadangaben bereitgestellen. Generell ist es bei Pfadangaben wichtig auf die richtige Groß- und Kleinschreibung zu achten. Beispiele für Pfadangaben:



Bei den meisten Scripts braucht man irgendwo einen Basedirectory-Pfad
und genau da entstehen (und nicht nur bei Exit) die meisten Fehler.
Auf Exit muss er wie folgt aussehen:
/usr/local/apache/htdocs/clans/x/member/y

x = der Name deines Clans KLEINGESCHRIEBEN
y = dein Username mit richtiger Groß-Kleinschreibung

dahinter können auch noch Unterverzeichnisse von dir kommen.
Das wichtigste dabei ist aber überall die richtige Schreibung, denn Exit
benützt einen Linux-Server und Linux-Server unterscheiden zwischen
Groß- und Kleinschreibung:
script.pl ist also eine andere Datei als Script.pl


Deshalb ist das auch allgemein bei Pfadangabe im Script zu beachten.

Upload:


Scripts muss man mit einem FTP-Programm uploaden,
dass man sich aus dem Netz ziehen kann: (Empfohlen: WS_FTP)

Windows:
WS_FTP
Cute FTP
Macintosh:
Anarchie
OS/2:
Gibbon FTP
Unix:
LLNL XFTP

Der Host für CGI-Scripte auf Exit muss wie folgt lauten:
cgi.exit.de

CGI-Scripts müssen in ASCII formatiert upgeloaded werden!
Außerdem müssen sie auf den meisten Servern auf den chmod 755 gestellt werden!

chmod 755 ... :


chmod ist ein Unixbefehl.
Damit werden die Zugriffsrechte einer Datei vergeben.

Manuel kann der Befehl zum Beispiel heißen:
quote site chmod 755 datei.cgi/pl

Mit Programmen wie WS_FTP oder CuteFTP, die ich an dieser stelle
zu empfehlen sind, geht das ganze aber viel leichter.

Bei WS_FTP muss man nur mit der rechten Maustaste auf das Script und dann auf chmod klicken.

Dann kann man die Zugriffsrechte einstellen (folgende Kombi steht für 755):

xxx
Internal Server Error 500:


Wenn dieser Fehler erscheint, dann wurde immerhin ein Teil richtig gemacht!

Als erstes sollte man das Script noch zweimal genau überprüfen.

Häufige Fehler:

  1. das Zeichen " wurde im HTML-Ausgabeteil des Scripts nicht durch '
    ersetzt oder es wurde vergessen vergessen \ davor zu setzen!
  2. im Quellcode wurde eine Klammer vergessen oder ein Semikolon!
  3. oder man hat zum Beispiel ausversehen eine unendliche Schleife geschrieben!

Dies sind nur ein paar mögliche Fehler, die einem schon mal unterlaufen können.
Wenn man den Fehler nicht findet, sollte man im Kompendium von Stefan Münz
nachschauen was ihr falsch gemacht habt!
